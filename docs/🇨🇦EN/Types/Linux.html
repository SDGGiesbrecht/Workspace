Error: Swift Is Unavailable

Because the script opens a new terminal to display its output, brand‚Äênew terminal sessions need to be able to find the swift command without additional set‚Äêup.

To register the location of swift even for new terminal sessions, run the following command, substituting the real location of the Swift install.

(If Swift is not even istalled yet, see the Swift website for instructions.)

Running from a Terminal

While double‚Äêclicking is usually the most convenient, it can be bypassed by manually running the macOS equivalent from a terminal:

This is exactly what the Linux script does internally. It just opens a new terminal window first to display the output.
